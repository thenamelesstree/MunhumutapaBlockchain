Digital Certificates

Introduction

The digital certificates distributed application is a public certificates storage, authentication and verification system secured by the blockchain network.

The system allows the authentication of digital certificates through key generation and subsequent storage of certificate data through Munhumutapa accounts.

This certification system brings trust and transparency to digital certification as it allows institutions and users to authenticate and verify certificates issued in digital format to a particular recipient e.g. individuals, organizations, products or services.

The system also stores valuable data including dates of issue, renewals and recipient Munhumutapa accounts.

Certificate Properties

Name

A certificate's name can be used to align it to a group of similar certificates, e.g. certification standards, industry, scope etc in respect of its issuer and recipients.

Type

A certificate type highlights the sector under which it falls in relation to its name and description e.g. educational certificates, standards certification, professional certification etc.

Certificate Description

The description of a certificate gives information on what the certificate represents and/or to whom it has to been issued to e.g. name of individual, organization, product etc. Sets of certificates may therefore have the same description to indicate similarity.

Recipient

Every certificate is issued to a specific Munhumutapa account from an equally distinct issuer account. This relationship allows users to verify the authenticity if each certificate by estabilishing that an authorised account indeed certified another account.
